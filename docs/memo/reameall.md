## 🚀 プロジェクト概要：ZeroSecondThink

「0秒思考」をシステムで補助し、思考の言語化・習慣化・高度化を支援するアプリ。



## 🛠 技術スタック

| カテゴリ | 選定技術 | 備考 |
| --- | --- | --- |
| **Frontend** | Next.js, Vercel | 高速なUI/UXとデプロイ |
| **Backend** | FastAPI, Cloud Run | PythonによるAI連携の容易さ |
| **Database** | Supabase (PostgreSQL) | Auth, DB, Strageの一括管理 |
| **Database** | Turso (SQLite) | 一次的なログの保存場所 |
| **AI Model** | Gemini (Google AI Studio) | 文面精査、タグ生成、週次分析 |

## 要件

###　機能要件
- ログインできること
  - Googleでのログインも可能とする
- １日１０件ぐらいのメモ（ポスト）が可能なこと（最大で１日２０件）
- ポストごとに、タグを付与可能なこと
  - ただし、タグは文面から生成AIでの作成も可能
- 「ダッシュボード画面」をもつこと
- 「ダッシュボード画面」では、登録の件数などをチャート化して、習慣化できているかなどを表示する
- 「分析画面」をもつこと
- 生成AIを利用して「文面・タグの精査」ができること
  - 生成AIでの文面の「文面・タグの精査」は１週間単位で実施可能
    - 一度に２０件で、１週間７０件だと３回の生成AIへのリクエストを想定
  - １時間に１回の未精査のポスト（記事）については自動精査をするものとする。
  - 精査済の物は精査内容確認しての登録と再精査の要求ができること
  - 生成AIで精査後のものについてはフラグを持つもとする。
  - 精査後のコンテンツに関しては、別途メールで連携すること
- 「タグメンテナンス画面」適切なタグ付のために、AIと共有すべき、タグについては別にメンテナンスできること。
  - 「タグメンテナンス画面」ではタグとして表示するときの名前、名前、別名（エイリアス）、説明などを登録できる。
  - 「タグメンテナンス画面」で登録したものは生成AIへの「文面・タグの精査」依頼時に情報として送信する。
- 登録した記事はパブリックがOFFの状態で、生成AIで文面精査後にONにできる。
-「週単位での評価」をAIに依頼できるようにする。
  - 「週単位での評価」依頼時は、1週間分のポストの中でパブリックなものを対象とする
  - 「週単位での評価」依頼時のAIの返答とそれに対する変更、更新は可能とする。
  - 「週単位での評価」依頼時には表対象週の前週、前々週も過去の情報として送信する。
- 「週単位での評価」は自動でAIへの依頼を行い、結果をメールなど連絡し、「週単位での評価画面」での登録を促す
- 「週単位での評価画面」は週単位での評価などを行うものとする。
- 「週単位での評価画面」で登録されたものは、別画面で参照可能、そのまま、当日のポストとして登録可能にする。
- 「分析画面」ではタグごとの件数の推移（週単位）や、タグの記載内容の一覧を確認できて、今後の思考パターンにつなげる。

## 📋 詳細要件

### 1. メモ・精査サイクル

* **投稿制限:** 1日10〜20件程度のクイックなアウトプット。
* **AI自動精査:** 1時間ごとのバッチ処理。未精査のポストをAIが分析。
* **フラグ管理:** 「精査済み」「パブリック設定」の管理。
* **通知:** 精査完了後、別途メールで内容を送信。


* **再精査機能:** AIの回答に納得がいかない場合、フィードバックを与えて再実行が可能。

### 2. タグマネジメント

* **マスタ管理:** タグ名、別名、説明をユーザーが定義。
* **AI連携:** 定義したタグ情報をAIにコンテキストとして渡し、精度の高い自動分類を実現。

### 3. 分析・評価（習慣化サポート）

* **ダッシュボード:** 投稿件数の推移をチャート表示。
* **週次AI評価:** 直近3週間分（対象週＋前2週）のデータを元に、思考の深まりを分析。
* 評価結果はメール通知し、専用画面で確認・ポストへの再登録が可能。

「自動精査（1時間に1回）」と「手動での再精査要求」のフローが複雑になりがちです。以下のようなステータス管理のイメージを共有すると、実装ミスが減ります。
> **ポストの状態遷移：** 未精査 → AI自動精査中 → 精査済（要確認） → ユーザー確認・修正 → パブリックON（確定）
> **「パブリックON」の定義:** １週間のまとめについて利用するものを想定しています。パブリックOFF＝プライベートなので、公表するものには含めない想定です。
